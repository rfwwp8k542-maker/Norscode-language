# Frontend Browser E2E

Browser-E2E tester hele flyten slik en bruker opplever den.

## Mål

- verifisere ekte brukerreise i browser
- sikre at routing, data og skjemaer fungerer sammen
- fange feil som komponenttester ikke ser

## Modell

- test representative brukerscenarier i browser
- verifiser navigasjon, submit og tilbakemelding
- test både normalflyt og feilsituasjoner

## Regler

- hold E2E-testene få og viktige
- test hele reisen, ikke alle små detaljer
- bruk E2E til det som virkelig krever browser

## Når dette er ferdig

- de viktigste brukerflytene er automatisk verifisert i browser
