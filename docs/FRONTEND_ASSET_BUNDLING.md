# Frontend Asset Bundling

Frontend bør ha en enkel og forutsigbar strategi for statiske filer.

## Mål

- samle CSS, bilder og ikoner på en ryddig måte
- gjøre publisering enkel
- unngå unødvendig byggekompleksitet i første versjon

## Modell

- start med enkel statisk publisering
- bruk bundling bare når det faktisk gir verdi
- hold asset-navn stabile

## Regler

- CSS skal være lett å finne
- bilder og ikoner skal være separate og oversiktlige
- statiske filer bør kunne caches hardt

## Når dette er ferdig

- frontend-ressurser kan bygges og publiseres forutsigbart
