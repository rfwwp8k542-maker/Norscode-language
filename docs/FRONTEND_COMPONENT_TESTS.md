# Frontend Component Tests

Komponenttester bør være den første testtypen for frontend.

## Mål

- verifisere komponenter i isolasjon
- oppdage regresjoner tidlig
- holde testene raske og enkle

## Modell

- test komponenter med representative props
- test rendering av viktig markup
- test tom- og feiltilstander
- test at standardkomponenter oppfører seg konsistent

## Regler

- test det som faktisk betyr noe for brukeren
- hold testene små og lesbare
- unngå å teste implementasjonsdetaljer unødvendig

## Når dette er ferdig

- komponentbiblioteket har trygg dekning
- endringer i UI blir lettere å gjøre uten å bryte alt
