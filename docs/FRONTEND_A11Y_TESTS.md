# Frontend Accessibility Checks

Accessibility-sjekker bør være en fast del av frontend-kvaliteten.

## Mål

- oppdage manglende labels og dårlig kontrast
- sikre keyboard-tilgang
- fange problemer som visuelle tester ikke ser

## Modell

- sjekk semantisk HTML
- sjekk labels, fokus og kontrast
- verifiser at viktige widgets kan brukes med tastatur

## Regler

- tilgjengelighet er ikke et tillegg
- test det som faktisk bryter brukeropplevelsen
- hold sjekkene enkle og gjentakbare

## Når dette er ferdig

- viktige a11y-problemer oppdages automatisk
- frontend er mer tilgjengelig for flere brukere
