# Frontend Accessibility

Tilgjengelighet bør være en standarddel av frontend-designet.

## Mål

- gjøre appen brukbar med tastatur
- gjøre innhold lesbart for skjermlesere
- holde kontrast og fokus tydelig

## Modell

- bruk semantisk HTML når det er mulig
- gi knapper og inputs tydelige labels
- sørg for fokusmarkering
- ikke bruk farge alene for å formidle mening

## Regler

- alt interaktivt må kunne nås med tastatur
- feilmeldinger må kunne oppdages av hjelpemidler
- kontrast må være lesbar i begge temaer

## Når dette er ferdig

- appen fungerer for flere brukergrupper
- tilgjengelighet er en del av standardkomponentene
